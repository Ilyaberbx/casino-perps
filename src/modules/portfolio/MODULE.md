# portfolio (client)

> **Scope of this file:** non-obvious context only. If something is obvious from `index.ts` or by reading a few files, do NOT add it here. Update this file in the same PR that changes the module's public surface, the things it owns (tables/routes/providers), or its dependencies on other modules.

## Purpose

The portfolio **Page** — three-column top region (left tile column / center summary card / right chart card) + bottom tab panel. Reads portfolio data from the active **Venue** via the `PortfolioReader` capability (narrows on `useVenue().capabilities.portfolio`). See `CONTEXT.md` for `Account Value`, `PNL`, `Window`, `Snapshot`, `History`; ADR-0008 for the capability composition.

## Public surface

- `PortfolioPage` — the route-addressable page (assembled by `app/`).

## Owns

- Page composition and its hook (`use-portfolio-page`).
- Internal sub-components — **not exported**; consumed only by `PortfolioPage`:
  - `portfolio-chart` — smooth (monotone-cubic) line chart using chart.js + react-chartjs-2; the sparse venue history is resampled to a dense, time-uniform series via `densifyPortfolioPoints` (target `SMOOTH_TARGET_POINTS`) so the line curves instead of stepping and the X ticks land at even clock-time intervals; takes `metric` + `window` props for axis/tooltip formatting; renders visible X/Y tick labels (window-aware on X, sparse on Y) via `formatTickLabel`, with vertical grid lines aligned to the labeled ticks (`pickXTickIndices`); tooltip title uses `formatTooltipTitle`; tooltip body uses `formatPortfolioValue`; four local chart.js plugins live in `portfolio-chart.plugins.ts` — `portfolioZeroBaseline` (dashed `y=0` rule, only when `metric` is signed: `pnl` / `perpsPnl`), `portfolioLineGlow` (re-strokes the rendered polyline with a canvas shadow in the series tone — `shadowBlur` + tone `shadowColor` — to bloom the line only, leaving the gradient fill's edge crisp; traces only finite points so the glow honors the entry sweep), `portfolioCrosshair` (full crosshair: thin vertical + horizontal hairlines at the hovered point, a tone-colored "lit" halo, and a crisp square hover point), `portfolioEndValuePin` (last-value drawn as a bordered price-tag chip in the reserved right pad, hidden while tooltip is active) — array order is draw order, so the glow sits under the interaction chrome; animation is a 600ms left-to-right draw-on on initial draw (each dense sample revealed on a per-point delay, x from NaN, y from the previous point's pixel, so the line + `fill: 'origin'` area sweep in left→right; `ENTRY_DURATION_MS`), collapsed to 0 under `prefers-reduced-motion` via `prefersReducedMotion` (the canvas animation bypasses the global CSS kill switch), instant on tooltip/resize transitions; `<Line>` is keyed on `${metric}-${window}` so a metric/window switch hard-remounts and replays the entry animation
  - `portfolio-chart-card/` — right card with Chart metric switcher (Account Value | PNL | Perps PNL); drives `getHistory(metric, window, scope)`; renders a period-delta badge in the header (right-aligned, tone-colored by sign, hidden when the series is empty/flat) computed via `computePeriodDelta` + `formatPeriodDelta`
  - `portfolio-summary-card/` — center **Account Equity** card, restyled to the #275 reference: a header with the Accounts (All | Only Perps) scope control + Period (24H | 7D | 30D | All-Time) control, then six flat `StatRow`s in fixed order — **PNL, Volume, Max Drawdown, Total Equity, Perp Account Equity, Spot Account Equity**. PNL / Volume are selected by the active Period from the window-keyed `snapshot.pnl[window]` / `snapshot.volume[window]` (ADR-0039); Total Equity = `snapshot.accountValue`, Perp/Spot Account Equity = `snapshot.perpsEquity` / `snapshot.spotEquity`. **Max Drawdown is a placeholder dash** — Hyperliquid does not surface it and equity facets are never app-computed (CONTEXT.md), so the row renders `—` (never a client calculation). The uPnL decomposition + the `equityExtensions` buckets that used to render here are gone (the detailed Vault / Earn / Staking breakdown lives on the Trade-page `trade-equity-card`). The colocated `use-portfolio-summary-card` hook owns the Pro/Simple control-style split (`useIsSimpleMode`, #276): Pro renders `SegmentedControl`s, Simple renders `IconSelect` dropdowns (same options + selection). Takes an `isSegregated` prop (owned by `use-portfolio-page`, fed from `venue.capabilities.accountMode`): when `false` (unified / portfolio-margin) the `'Only Perps'` scope option is hidden via `visibleScopeOptions` — a unified account shows only the combined view (ADR-0033 D-4). The page hook also snaps the active scope back to `'all'` when it becomes unified, so the reader is never subscribed for a phantom perps scope.
  - `portfolio-tile-column/` — left column: mounts the venue-independent `<AgentBalanceTile />` (from `@/modules/agent-balance`) at the top, then `VolumeTile` reads `PortfolioSnapshot.fourteenDayVolume` via `PortfolioReader` and opens `VolumeHistoryModal` on "View Volume"; `FeesTile` reads `FeeScheduleReader` and opens `FeeScheduleModal` on "View Fee Schedule" (explicit unsupported state when capability absent). The Agent Balance tile is structurally separate from the venue/portfolio tiles — it reads Base on-chain USDC, not the active Venue.
  - `volume-history-modal/` — modal showing `VolumeHistory.entries` (date + exchange/maker/taker USD) plus a Total row summed across entries; subscribes via `useVenue().capabilities.volumeHistory`
  - `fee-schedule-modal/` — modal with four read-only sections (Referral Discount, Staking Discount, Maker Rebate, Volume Tier); Volume Tier table toggles Spot/Perps and highlights the user's active row by matching `userSpotTakerRate` / `userPerpsTakerRate` against the tier rate; subscribes via `useVenue().capabilities.feeSchedule`
  - Bottom region now mounts the shared `<AccountDock />` (`shared/components/account-dock/`). The portfolio-internal `portfolio-tab-panel/` and its per-tab hooks were removed in favour of the unified dock; both Portfolio and Trade pages now share the same nine-tab implementation, capability gating, paged readers, and decimal-formatting policy.
  - `portfolio-summary`, `window-selector` — legacy sub-components (kept for internal use). `PortfolioSummary` carries no Period selector, so it pins the `'24H'` bucket of the now-window-keyed `snapshot.pnl` / `snapshot.volume` (ADR-0039) — the value it always showed.

- `agent-balance/` — `AgentBalanceTile`, mounted at the top of `portfolio-tile-column/`. The only public-surface consumer of the agent-balance module; the tile owns its own Base RPC read and wallet gate, so the portfolio page passes no props.
- `trading/` — `MobileTradeDock` + `TradeDockProviders` (from `@/modules/trading`), rendered by `PortfolioPage` **on mobile only** (`useIsMobile`) as a sibling of the page `ScrollArea`. The page reserves bottom space for the fixed footer via `.shell.shellMobile`. The global header nav is hidden on `/portfolio` mobile (`AppShell`'s `hideHeader`), so the footer is the sole nav surface there. This is the module's only dependency on `trading/`; mounting the dock pulls in order entry + the AI sheets, but portfolio neither owns nor configures those flows.

## Cross-app contract

None — portfolio data flows through the `PortfolioReader` capability to a Venue, not the backend.

## Gotchas

- Same venue-coupling rules as `trading/`: never import a venue module directly. The `PortfolioReader` capability is the only allowed surface; if a venue does not advertise it, the page renders an explicit unsupported state.
- A venue switch hard-remounts the page — don't persist window/state across venues unless it is part of `shared/` user preferences.

## Out of scope

- Order entry, orderbook, charts of price (lives in `trading/`).
- Identity (lives in `account/`).
- Persisting portfolio history server-side (no backend route today; data is venue-derived).
